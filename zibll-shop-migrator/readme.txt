安全迁移 Zibll 商城商品、完整多值 Meta、分类层级、特色图、正文图片和 Meta 图片。

== 主要修复 ==

* 保留同一 Meta 字段的全部值，不再只取第一个值。
* 排除旧站特色图 ID 等系统字段，避免附件 ID 污染新站。
* 迁移特色图、正文图片、响应式图片、懒加载图片和图片类 Meta 中的附件。
* 安全解码序列化数组，拒绝导入 PHP 对象，避免对象注入。
* 商品写入使用 WP_Error 模式，失败时不再误报成功。
* 不再按标题查重，改用来源网站与原商品 ID，并以商品别名作为后备匹配。
* 保留分类名称、别名、描述和父子层级。
* 加入选择、搜索、进度、导出完成提示和详细失败报告。

== 使用方法 ==

1. 在旧站安装插件，进入“工具 → Zibll 商城迁移”。
2. 选择需要迁移的数据和商品，下载 ZIP 迁移包。
3. 备份新站数据库和 uploads 目录。
4. 在新站安装同一插件并上传 ZIP。
5. 建议勾选覆盖和保留原状态，然后开始导入。
6. 检查导入报告及商品价格、库存、属性、图片和分类。

== 注意 ==

* 新旧网站应使用兼容版本的 Zibll 商城数据结构。
* 商品 Meta 可能含敏感购买配置，请勿公开分享迁移包。
* 图片很多时迁移包会很大，需要足够的上传限制、内存和执行时间。
* 插件不迁移订单、用户余额、支付记录或用户账号。
